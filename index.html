What is Git?

It is like a seat beltâ€“you feel unsafe without it.

Git is a free and open source distributed version control system. It is also known as a source code management system. It allows you to track changes in your code so that previous versions are readily available. Git runs locally.

Not only does git give you version control, but it also allows you to work on components of your project with git branch . The branches can all be merged together at a later time with git merge .

Git for personal use

  1. Make a git repository for a project.
  2. Make a change to the project and commit the change.
  3. Make a branch for the project.
  4. Merge the branch to the master branch.

git init.

Creating a repository

  • Create a folder titled "git_practice."
  • Create a git repository.
  • git init

git commit

Taking a snapshot of your work history

  • Create a text file called "myfile.txt" and write some comments inside of it.
  • Add this work to the git repository by a git commit.
  • git add myfile.txt
  • git commit -m "my first commit"

What happens if you change things in the repository but you do not commit these changes?

Git will know.

  • Make a change to "myfile.txt" in some way.
  • Check the status of your git log.
  • git status
  • Take a look at the changes made.
  • git diff
  • Add and commit your changes.

git branch

Makes a branch from your current branch

  • Create a new branch for Section 1.
  • git branch section1
  • Not much has appeared to happen, you are still on the master branch.
  • git checkout section1
  • While in branch section1, add some changes to myfile.txt pertaining to section1.
  • Add and commit your changes.
  • Go back to the master branch.
  • git checkout master
  • What do you notice in myfile.txt?

git merge

Merges a branch to your current branch.

  • We will merge our section1 branch with the master branch.
  • git merge section1

git clone

Downloads a project with entire work history from the remote repository on github.

  • Create a local MATH110OER repository from the remote one on github.

Branch, change, merge.

  1. We will make a branch on the section we want to change.
  2. Add something to that section.
  3. Go back to the master branch and merge the branch to master.
  4. Pull the information from the current repository.
  5. Push the changes you made locally onto the remote repository.

Make a branch and merge to master

  • git checkout -b < name of your branch >
  • Make some changes to the particular section of your choice.
  • git add .
  • git commit -am "message of your choice"
  • git checkout master
  • git merge < name of your branch >

git pull

Fetches the changes from the remote and merges the changes to your local branch.

git pull

git push

Updates the remote repository with the changes on your local repository

git push origin master
